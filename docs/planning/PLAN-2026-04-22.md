# Plano de revisão e limpeza — Sequência Viral

**Data:** 2026-04-22
**Contexto:** Projeto tá funcional mas bagunçado. Muita variante de landing, onboarding de 2873 linhas, settings de 2973 linhas, rotas API mortas, FB Login acabou de entrar em paralelo ao Apify. Dono pediu: manter Apify ativo, revisar tudo, avaliar API alternativa, organizar.

## Diagnóstico

### O que está funcional hoje

- **Onboarding** Apify → image cache Supabase → vision transcripts → brand analysis → 3 carrosséis reais → redirect pra gallery. Confirmado 200 em prod (`viral.kaleidos.com.br/app/onboarding`).
- **Privacy / terms / data-deletion** live pra Meta review.
- **Endpoints Meta**: `/api/auth/meta/exchange`, `/api/auth/meta/callback` (stub 302), `/api/meta/deletion-callback` (HMAC-validated).
- **FB Login opt-in** no step connect do onboarding (só aparece se `NEXT_PUBLIC_FACEBOOK_APP_ID` setado — Gabriel adicionou).
- **Supabase**: tabelas `meta_connections` + `meta_deletion_requests` rodadas.

### O que está bagunçado

1. **Onboarding monolítico** — `app/app/onboarding/page.tsx` tem 2873 linhas. Todos os steps, estados, sub-componentes num arquivo só. Dificulta debug e aumenta risco de regressão.
2. **Settings monolítico** — `app/app/settings/page.tsx` tem 2973 linhas. Mesma história.
3. **Dead code**:
   - `app/app/onboarding-v1/` (1921 linhas, antigo, não importado)
   - `app/app/onboarding-v2/` (pasta vazia)
   - `app/app/create-v2/page.tsx` (redirect inútil, duplica `create/page.tsx`)
   - `app/api/generate-v2/` (7 linhas, zero refs)
   - `components/landing/gallery-section.tsx` (0 refs)
   - `components/landing/manifesto.tsx` (0 refs)
4. **Landing variants** — 5 rotas (`/landing/v2..v5`, `chatsheet`). Decisão: manter como hub de comparação (já documentadas em `/landing/page.tsx`), **mas** deletar packages órfãos `packages/brillance-landing/`, `nexus-landing/`, `optimus-landing/` (5.6MB de código não renderizado em nenhuma rota).
5. **Sem fallback de scraper** — se Apify cair ou zerar créditos, onboarding morre. Precisa fallback.
6. **README desatualizado** — não menciona FB Login, não explica múltiplas landings.

### Fase 2 — Fallback de scraper (esta sessão, ~45 min)

**Escolha: ScrapeCreators** como fallback do Apify. Razão:
- Mesmo modelo pay-as-you-go ($10 = 5k calls, ~$0.002/call)
- 100 calls free sem cartão (fácil validar)
- Retorna tudo: profile, últimos posts, carousel children, image URLs
- Integração REST simples — sem SDK exótico

**Estratégia**: circuit breaker. Tenta Apify primeiro. Se erro ou timeout >15s, cai automaticamente pro ScrapeCreators. Log qual foi usado.
