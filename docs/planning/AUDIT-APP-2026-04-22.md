## Parte A — Fluxo de onboarding

Arquivo: `app/app/onboarding/page.tsx` (3.147 linhas, client component, state machine `Step` linear).

Máquina de steps: `about → connect → analyze → dna → photo → visual → ideas → generating → done`.
Guard rígido: se `profile.onboarding_completed` é `true`, redirect pra `/app` (evita re-rodar grátis).

### Passo 1 · about
- Coleta: `displayName` (mín. 2 chars), `whatsapp` (opcional).
- Sem API. Hidrata a partir de `profile` / `user.user_metadata.full_name`.
- Sem falhas possíveis — só valida comprimento.

### Passo 2 · connect
- User digita `@handle` do Instagram **ou** clica em "Entrar com Facebook" (`<FacebookLoginButton>`).
- Se veio do Facebook, o handle vem como `preScraped` já pronto — pula o passo 3.
- Decisão do user: IG via scrape OU IG Business via Graph API do Meta.

### Passo 3 · analyze
- Pipeline em 6 fases (UI mostra progresso `analyzePhase`):
  1. scrape → `POST /api/profile-scraper` (Apify Instagram actor) → retorna bio, followers, 20 posts recentes com `imageUrl`, `slideUrls`, flag `isCarousel`.
  2. vision → `POST /api/post-transcripts` (Gemini 2.5 Flash inline image) transcreve os 8 primeiros slides pra capturar texto visível.
  3. brand analysis → `POST /api/brand-analysis` roda Gemini Pro com bio + posts + transcripts. Devolve `detected_niche[]`, `tone_detected`, `top_topics`, `posting_frequency`, `avg_engagement`, `suggested_pillars`, `suggested_audience`, `who_you_are`, `communication_style`.
- Duração: 20–45s (depende de peso do Apify + Gemini).
- Falhas: scrape privado/404 → erro amigável "Não consegui ler esse perfil"; sem posts → pula vision direto pra DNA em branco; vision falha → segue só com text (best-effort); brand-analysis falha → erro hard com CTA retry.
- Persiste: nada ainda — tudo em state local (`scrapedProfile`, `brandAnalysis`).

### Passo 4 · dna
- Form editável populado pela brand-analysis: `dnaNiches[]` (chips add/remove), `dnaTone` (select 4 opções), `dnaWho` (3–5 frases), `dnaAudience`, `dnaStyle`, `dnaPillars` (csv).
- Sem API. User pode ajustar tudo que a IA adivinhou.

### Passo 5 · photo
- Upload de avatar (`<input type=file>` → FileReader → `POST /api/upload` com `carouselId=profile`).
- Limite 5MB. Salva em Supabase Storage bucket `carousel-images` e retorna URL pública.
- Se user pulou, o avatar do IG scrapeado vira fallback.

### Passo 6 · visual
- User escolhe: `brandColorId` (6 swatches hex + custom hex input), `imageStyle` (photo/illus/iso3d), `designId` (template Futurista "manifesto" ou Twitter).
- Sem API. Só state local + previews inline.

### Passo 7 · ideas
- Ao entrar no step chama `regenIdeas()`: `updateProfile({ niche, tone })` + `GET /api/suggestions?refresh=1`.
- API chama Gemini Pro com grounding no Google Search pra ideias fresquinhas do nicho/persona. Cache 24h default.
- User marca até 3 ideias aprovadas (`approvedIdeas`). Pode regenerar lote.
- Falha → lista fica vazia, user segue sem selecionar (gera só com topic).

### Passo 8 · generating
- `saveProfileBeforeGeneration()` persiste o perfil final (`brand_analysis`, `niche`, `tone`, `carousel_style`, `brand_colors`, `onboarding_completed=true`) via `updateProfile`.
- Pré-aloca 3 UUIDs pra evitar duplicata em retry.
- Loop **serial** (não paralelo, pra evitar race na quota) de 3 `POST /api/generate` com `topic=idea.title`, `designTemplate=designId`.
- Cada sucesso chama `upsertUserCarousel(supabase, user.id, {...})` salvando slides em `carousels` com `status=draft`, `creationMode=quick`, `accentOverride=colorHex`.
- UI mostra progresso por item (`queued | running | done | error`). Erro isolado em um carrossel não trava os outros.

### Passo 9 · done
- Tela de confirmação. Link pra dashboard (`/app`).
- Dados persistidos: `profile` completo + 1–3 rows em `carousels`.

## Parte B — Criação de carrossel

Rotas em `app/app/create/`. Fluxo moderno pula `/concepts` (legado) e vai direto brief → slides com imagens → editor.

### Tela 1 · /app/create/new
`app/app/create/new/page.tsx` (1.726 linhas).
- Form principal: textarea `idea` (brief livre até 4.900 chars), cycler `tone` (editorial/informal/direto/provocativo), cycler `lang` (pt-br/en), cycler `niche`, picker `designTemplate` (manifesto/twitter), toggle `mode` (writer/layout-only).
- 5 shortcuts pré-preenchidos (tutorial, hot take, youtube, remix, case).
- Bloco "Avançado" opcional: `hookDirection`, `customCta`, `numSlides` (6–12), `extraContext`, `uploadedImageUrls` (upload direto via `/api/upload`).
- Interview mode (opcional, setting): faz perguntas antes de gerar.
- Fluxo ao clicar "Gerar":
  1. `upsertUserCarousel` com draft vazio pra pegar id estável.
  2. `detectSource(idea)` detecta YouTube/link/IG/idea.
  3. `generateCarousel()` chama `POST /api/generate` (lib `use-generate.ts`).
  4. Fase `images`: pra cada slide que veio sem `imageUrl`, `POST /api/images` com `mode=search` (twitter) ou `mode=generate` (manifesto + cover → Imagen 2-pass). Paralelo com `AbortSignal.timeout` 12–75s.
  5. Fase `finalizing`: `upsertUserCarousel` final com slides + images + `visualTemplate` + `promptUsed`.
  6. Redirect → `/app/create/{id}/edit?template={templateId}` (pula /templates).
- Erros: `PLAN_LIMIT_REACHED` → envia pra `/plans`; `429` → mostra retry-after; `503` → IA offline.

### Tela 2 (opcional) · /app/create/[id]/concepts
`concepts/page.tsx` (748 linhas) — fluxo **legado**. Chama `generateConcepts` que devolve 5 ângulos, user escolhe um, gera carrossel. Novo fluxo pula isso, mas a URL ainda funciona pra testes manuais e pra rascunhos antigos que começaram aqui.

### Tela 3 · /app/create/[id]/templates
`templates/page.tsx` (440 linhas).
- Grid 2×2 com preview REAL dos 4 templates (`manifesto`, `twitter`, `futurista` e `autoral` — últimos dois legado/escondidos).
- Usa `<TemplateRenderer>` com slides do draft — é literalmente o editor em miniatura.
- `fillVariants()` reatribui variantes narrativas (cover/solid-brand/full-photo-bottom/text-only/cta) quando o draft vem sem `variant`.
- Ao selecionar: `upsertUserCarousel({ visualTemplate })` + redirect `/edit`.

### Tela 4 · /app/create/[id]/edit
`edit/page.tsx` (1.990 linhas, coração do editor).
- Layout: 3 colunas desktop / tabs mobile. Sidebar esquerda (variantes por slide + layers texto), canvas central (`<TemplateRenderer>` 1080×1350 com scale), sidebar direita (branding: accent color, display font, textScale).
- `useDraft(id)` carrega estado; `useAutoSaveDraft(draft, 1200)` persiste via `carousel-storage.upsertUserCarousel` a cada 1,2s.
- Canvas edita `heading`, `body`, `imageQuery`, `variant`, `layers` inline.
- `<ImagePicker>` + `useImages` → `POST /api/images` com `count=8` retorna opções; user escolhe uma. Upload manual via `/api/upload` também suportado.
- Fontes display injetadas via `<link>` dinâmico (Inter/Archivo Black/Bebas Neue/Anton/Oswald/Barlow Condensed/Instrument Serif).
- CTA "Continuar" grava `sv_preview_snapshot_${id}` no sessionStorage (estratégia anti-race) e redireciona pra /preview.

### Tela 5 · /app/create/[id]/preview
`preview/page.tsx` (1.139 linhas).
- iPhone mockup com slides do **snapshot** do sessionStorage (fonte de verdade primária, fallback pro DB).
- Controles:
  - Export PNG (todos slides juntos em zip via `lib/export-zip.ts` com `html-to-image toPng`).
  - Export PDF (`jsPDF` — lazy import).
  - Copy to clipboard (só capa).
  - Marcar como publicado → muda `status`.
  - Legenda editável: `useCaption` chama IA pra sugerir caption com base nos slides.
  - Panel `<CarouselFeedbackPanel>` coleta thumb up/down + motivo.
- `use-export.ts` faz `waitForImagesInElement` (timeout 3s por img) e `waitUntilRefsReady` antes do render.

### Entrega final
O user termina com um `.zip` (ou `.pdf`) localmente. O carrossel também fica salvo em `/app/carousels` pra reuso.

## Parte C — Telas principais do app

### /app — Dashboard
`app/app/page.tsx` (1.043 linhas).
- Header com nome + plan badge + quota (`usage_count/usage_limit`).
- `loadCarousels()` via `fetchUserCarousels`. Separa `drafts` e `published`, mostra 4 recentes de cada como cards mini com `<EditorialSlide>` preview scale 0.22.
- Seção "Ideias sugeridas": `loadIdeas()` → `GET /api/suggestions` (cache 24h, grounding Google Search). Fallback: deck mock estático de 3 ideias.
- CTAs levam pra `/app/create/new?idea=...`.
- Empty state se `totalCarousels === 0`: banner "Comece aqui".
- `<DiscountPopup trigger="post-onboarding">` dispara só pra free + pós-onboarding, cooldown 7d.

### /app/carousels — Biblioteca
Lista completa de carrosséis do user. Filtros: `all | drafts | published | archived`, filtro por tag, search text, sort recent/oldest. Ações: duplicar, excluir (com confirm), editar tags, export em batch, copiar link. Bulk select com checkboxes. Usa `fetchUserCarousels`, `deleteUserCarousel`, `updateCarouselTags`, `upsertUserCarousel` (pra duplicar) e `bumpCarouselUsage`.

### /app/plans — Pricing
3 cards: Free (R$ 0), Creator (R$ 49/mês ou R$ 39,20 anual), Pro (R$ 97/mês ou R$ 77,60 anual). Toggle month/year. Valores em centavos BRL de `lib/pricing.ts`. CTA → `/app/checkout?plan=...`.

### /app/checkout — Stripe handoff
Formulário final com plan selecionado, anchor price riscado, preço promocional, features. Cria session Stripe via `/api/stripe/checkout` e redireciona pro hosted checkout. Success callback volta com `?session_id=...`.

### /app/settings — Configurações
`app/app/settings/page.tsx` (2.961 linhas, 6 abas ativas):
- **Perfil**: name, avatar, instagram_handle, twitter_handle, linkedin_url.
- **Branding padrão**: brand_colors, carousel_style, fontes favoritas.
- **Redes**: integrações Instagram/LinkedIn (OAuth futuro).
- **Voz IA**: 4 presets (editorial/direto/provocador/pessoal) + samples + `voice_samples[]` (ingest manual de posts do user). Chama `/api/voice-ingest` e `/api/voice-samples`.
- **Plano**: mostra plan atual, quota, portal Stripe (`/api/stripe/portal`), upgrade/downgrade.
- **Segurança**: troca senha, deletar conta, logout.

### /app/gallery — Galeria de imagens
Tabela `user_images`: todas as imagens geradas (Imagen) ou upadas. Filtros all/generated/uploaded. Reuso: usuário pode copiar URL e colar no editor. Ações: deletar, baixar.

## Parte D — Fluxo técnico

### Pipeline de geração (caminho crítico)

```
User brief (/app/create/new)
        │
        ▼
POST /api/generate   (route handler → lib/server/generate-carousel.ts)
        │
        ├── detectSource(idea) → {video | link | instagram | idea}
        │
        ├── extractContentFromUrl()  (lib/url-extractor.ts)
        ├── getYouTubeTranscript()   (lib/youtube-transcript.ts + Supadata)
        │   OR lib/server/instagram-scrapers/*  (Apify)
        │
        ├── extractSourceFacts() → NER Gemini Flash (lib/server/source-ner.ts)
        │    Retorna: summary[], people[], entities[], numbers[], quotes[], dates[].
        │    Faz o carrossel ficar factual em vez de parafraseado.
        │
        ├── Monta systemPrompt + userMessage (brand_analysis do profile + NER facts + brief)
        │
        ├── geminiWithRetry()   (gemini-retry.ts — backoff exp, fallback Flash se Pro falha)
        │    Gemini 2.5 Pro (writer) OR 2.5 Flash (layout-only)
        │
        ├── Normaliza slides (heading, body, imageQuery, variant)
        └── Injeta imageUrl nos slides 0..N-1 se user fez upload manual
```

Depois do writer, o cliente (page `/create/new`) orquestra imagens:

```
For each slide sem imageUrl:
    POST /api/images
        │
        ├── planSlideImages(totalSlides)  (lib/server/image-strategy.ts)
        │    cover → Imagen 4 (2-pass: cover-scene prompt → imagen)
        │    par → Serper (stock editorial)
        │    impar → Gemini 3.1 Flash Image (geração leve)
        │
        ├── Cache temático (image_theme_cache, TTL 30d, sha256(mode::query))
        │
        ├── Upload → Supabase Storage (bucket carousel-images)
        │
        └── Persist em user_images (source, query, url)
```

Custo médio por carrossel de 6 slides: ~R\$ 0,45 (1 Imagen + 3 Gemini Flash Image + 2 Serper).

### Persistência

- `profiles` — user profile + `brand_analysis` JSONB populado no onboarding, `onboarding_completed`, `plan`, `usage_count`, `usage_limit`, `niche[]`, `tone`, `carousel_style`, `brand_colors[]`, `voice_samples[]`.
- `carousels` — 1 row por carrossel. Inclui `slides` (JSONB array), `status`, `visualTemplate`, `designTemplate`, `creationMode`, `accentOverride`, `displayFont`, `textScale`, `promptUsed`, `tags[]`, `savedAt`.
- `user_images` — galeria unificada.
- `suggestions_cache` — ideias Gemini + Google Search, TTL 24h por user.
- `generation_logs` — 1 row por geração com tokens, custo, prompt, duração.
- `image_theme_cache` — dedup temático de imagens.
- `carousel_feedback` — thumbs-up/down + motivo.
- `scrape_cache` — cache de scrapes Apify (IG profile / IG post).

### Libs-chave

| Arquivo | Papel |
|---|---|
| `lib/server/generate-carousel.ts` | Orquestra source extraction + NER + Gemini writer. |
| `lib/server/source-ner.ts` | Extrai fatos estruturados via Gemini Flash antes do writer (anti-paráfrase). |
| `lib/server/image-strategy.ts` | Decide Imagen vs Serper vs Gemini Flash Image por slide. |
| `lib/server/gemini-retry.ts` | Backoff exponencial + fallback Pro → Flash. |
| `lib/carousel-storage.ts` | CRUD tipado de carrosséis no Supabase (fetch/upsert/delete/tags/usage/feedback). |

| `lib/create/use-images.ts` | Client hook pra `/api/images` (busca + geração). |
| `lib/create/use-export.ts` | Export PNG/PDF via `html-to-image` + timeout 3s por imagem. |
| `lib/create/use-draft.ts` | Auto-save 1200ms debounced. |
| `lib/create/use-caption.ts` | IA pra caption final pré-publicação. |
| `lib/server/auth.ts` | Guard admin (ADMIN_EMAILS hardcoded) + RLS helpers. |
| `lib/server/rate-limit.ts` | Rate limit por user/ip + quota por plan. |

### Flow de autenticação

`lib/auth-context.tsx` (React Context) expõe `user`, `session`, `profile`, `updateProfile`, `refreshProfile`. Todas as rotas `/app/*` ficam atrás de `<AuthGate>` (layout). API routes validam via `lib/server/auth.ts` (`authenticateRequest()` parseia Bearer do Supabase JWT).

## Observações / inconsistências encontradas

- **`/app/create/legacy/page.tsx`** — só faz `redirect('/app/create/new')`. Dead weight mínimo mas podia virar redirect em `next.config.ts` e sumir do bundle.
- **Aba "Notificações"** em `/app/settings` — todas as opções estão comentadas no array `TABS` mas o tipo `TabId` ainda lista `"notifications"`. Orphan type.
- **Templates "autoral" e "futurista" legados** — escondidos no picker (`TEMPLATE_ORDER = ["manifesto", "twitter"]`) mas o renderer e os nomes ainda existem (`TEMPLATE_DESC`, `TEMPLATE_NAME_OVERRIDE`). Drafts antigos salvos com esses IDs ainda renderizam — ok manter por compat.
- **`/app/create/[id]/concepts`** — fluxo legado. `/new` pula direto pra `/edit`. Rota continua funcional mas nenhum CTA aponta pra ela. Candidato a deprecation.
- **`ADMIN_EMAILS` hardcoded** em 3 lugares (`app/app/admin/page.tsx`, `app/app/admin/users/[id]/page.tsx`, `app/app/admin/batch-test/page.tsx`, `source-test/page.tsx`). Cliente-side duplica server-side guard — pode virar `lib/admin-emails.ts` exportado.
- **Interview mode** em `/create/new` está ativo mas depende de `profile.interview_enabled` (feature flag). Não documentei em `/help`.
- **`/app/onboarding-v1`** — pasta legada identificada. Tentativa de delete via `git rm` e `rm -rf` foi bloqueada pelo sandbox de permissões do Claude (ver reporte).
